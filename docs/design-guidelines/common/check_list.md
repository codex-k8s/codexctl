# Общий чек-лист перед PR

Используется как self-check перед открытием PR.

## Архитектура

- Изменения двигают код к целевой Ports and Adapters архитектуре.
- Use-case слой зависит от интерфейсов, а не от SDK/CLI-инструментов.
- Границы `cli`/`application`/`core`/`adapters` не размыты.

## Порты и реализации

- Контракты `Orchestrator` и `Repository` остаются платформенно-нейтральными.
- Реализация по умолчанию соответствует:
  - Kubernetes через `client-go`;
  - GitHub через `go-github`.
- Новые изменения не блокируют будущие адаптеры `docker-compose` и `gitlab`.

## Публичные контракты

- Изменения CLI-флагов/env задокументированы.
- Изменения формата `services.yaml` задокументированы.
- Если менялись reusable workflow, обновлены контракт и примеры wrapper-подключения.

## Внешние зависимости

- Интеграция с `yaml-mcp-server` и `telegram-approver` учтена корректно.
- Таймауты и fallback-поведение для внешних операций заданы явно.
- Секреты не появляются в коде, логах, примерах и документации.

## Документация

- Обновлены релевантные документы `docs/design-guidelines/**`.
- Обновлены `README.md` и `README_RU.md`, если изменился пользовательский сценарий.
- Для Go-изменений пройден `docs/design-guidelines/go/check_list.md`.
