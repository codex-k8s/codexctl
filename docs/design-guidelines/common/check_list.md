# Общий чек-лист перед PR

Используется как self-check перед открытием PR.

## Архитектура

- Изменения двигают код к целевой архитектуре, а не закрепляют технический долг.
- Границы слоев не размыты: CLI parsing, orchestration, core-правила и adapters разделены.
- Инфраструктурные side effects не размазаны по несвязанным модулям.

## Публичные контракты

- Изменения CLI-флагов/env задокументированы.
- Изменения формата `services.yaml` задокументированы.
- Если менялись reusable workflows, обновлены контракт и примеры wrapper-подключения.

## Внешние зависимости

- Учет интеграции с Kubernetes и GitHub корректен.
- Учет интеграции с `yaml-mcp-server` и `telegram-approver` корректен.
- Таймауты и fallback-поведение для внешних вызовов явно определены.

## Безопасность

- Секреты не зафиксированы в коде/документации/логах.
- Нет небезопасных shell-конструкций с пользовательским вводом.
- Логи содержат диагностику, но не чувствительные данные.

## Документация

- Обновлены релевантные документы в `docs/design-guidelines/**`.
- Обновлены `README.md` и `README_RU.md`, если изменился пользовательский сценарий.
- Для Go-изменений пройден `docs/design-guidelines/go/check_list.md`.
