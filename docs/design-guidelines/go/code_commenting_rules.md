# Правила комментариев в Go

## Общие требования

- Комментарий добавляется только там, где код неочевиден без дополнительного контекста.
- Комментарий объясняет "почему", а не дублирует "что".
- Комментарии на экспортируемые сущности обязательны и должны быть актуальными.

## Где комментарии обязательны

- Нетривиальные решения в orchestration flow.
- Ограничения внешних API и workaround-поведение.
- Временные технические компромиссы с явным направлением рефакторинга.

## Где комментарии вредны

- Очевидные строки кода.
- Устаревшие TODO без контекста.
- Док-блоки, противоречащие фактическому поведению.
