# Наблюдаемость

## Логи

- Логи структурированные, с единым форматом полей.
- Минимальный обязательный контекст для orchestration-step:
  - command/use-case;
  - env/namespace/slot (если применимо);
  - внешний target (repo, service, resource);
  - duration и outcome.
- Секреты и чувствительные payload в логах запрещены.

## Трассируемость сценариев

- Каждый крупный CLI-сценарий логически разбит на шаги.
- По логам должно быть возможно восстановить: входные параметры, порядок шагов, точку падения.
- Ошибки внешних вызовов должны быть привязаны к конкретной операции.

## Метрики и диагностика

Эталонное направление:

- ввести lightweight телеметрию по ключевым сценариям (count/success/failure/latency);
- отделить пользовательский вывод от диагностического.

Даже до внедрения метрик логи должны обеспечивать эквивалентную диагностируемость.
