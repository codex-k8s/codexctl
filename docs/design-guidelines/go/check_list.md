# Go чек-лист перед PR

## Архитектура

- Изменения не усиливают расхождение с целевой пакетной архитектурой.
- CLI, application/core и adapters не смешаны в одном модуле.
- Новые интеграционные точки реализованы через явный adapter-контракт.

## Надежность

- Для внешних операций определены timeout/retry/cancel semantics.
- Ошибки нормализуются и диагностируемы.
- Нет неуправляемых shell-вызовов с небезопасной интерполяцией.

## MCP и кросс-репо

- Изменения `codex.mcp.servers` валидно рендерятся в `config.toml`.
- Для approval-like tool задан адекватный `tool_timeout_sec`.
- Проверена совместимость с задокументированными контрактами `yaml-mcp-server` и `telegram-approver`.

## Качество кода

- Выполнены: `go fmt ./...`, `go vet ./...`, `go test ./...`.
- При изменениях публичного поведения обновлены `README.md` и `README_RU.md`.
- Документация в `docs/design-guidelines/**` актуализирована.
